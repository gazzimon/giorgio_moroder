02 — Oracle Output & Technical Spec

Execution-Ready Oracle Intelligence

The oracle does not return prices. It returns a multidimensional execution payload that agents and smart contracts use to make deterministic capital allocation decisions.

oracle-payload.json

On-Chain Output Format

Canonical format: int256[4] with 1e6 scale precision.

Index Field Description
0 fair_price Reference value for pricing, collateralization, and execution sizing. Unit: USD (1e6)
1 confidence Reliability assessment (0.0 a 1.0). Higher values indicate more deterministic execution environments.
2 max_size Capital sizing constraint. Maximum claim size the oracle authorizes given current risk parameters.
3 flags Machine-readable risk states as bitmask. Each bit represents a specific risk condition.

Confidence Computation

confidence = f(oracle_divergence, volatility, liquidity_depth)
  • oracle_divergence: Deviation between SEDA nodes
  • volatility: Price variance in time window
  • liquidity_depth: Market depth in underlying assets

Risk Flags (Bitmask)

Bit 0 VOLATILITY_ALERT
Bit 1 LIQUIDITY_STRESS
Bit 2 ORACLE_DIVERGENCE
Bit 3 CIRCUIT_BREAKER

This structure allows agents and smart contracts to decide if, when, and how much capital to deploy. Claims are only issued when intelligence authorizes it.

03 — Capital Engine

Synthetic Claims, Not Stablecoins

USX402 is not a stablecoin. No peg is assumed. No immediate redemption is promised.

USX402 represents a redeemable synthetic claim on protocol-managed capital deployed under a WCRO/USDC LP strategy. Burning USX402 triggers settlement in USDC, subject to the liquidity state and risk controls enforced by the Capital Engine.

The protocol does not guarantee:

  • 1:1 backing at all times
  • Immediate redemption on demand
  • Fixed redemption rates

What the protocol does guarantee:

  • Deterministic issuance based on verified oracle output
  • Transparent capital allocation rules
  • Composability with external agents and protocols
